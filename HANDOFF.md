# 🤝 سند انتقال پروژه — IronZone.ir (سیستم مدیریت باشگاه بدنسازی)

این فایل برای این نوشته شده که اگر یک گفتگوی جدید با Claude شروع کردی، با خوندن همین یک فایل
کامل بفهمه پروژه چیه، چه معماری داره، چه محدودیت‌هایی داره، چی کار شده و چی باقی مونده.

---

## ۱. پروژه چیه؟

یک سایت مدیریت باشگاه بدنسازی به اسم **IronZone.ir** با:
- صفحه‌ی معرفی عمومی (Landing Page)
- صفحه‌ی ورود/ثبت‌نام کاربر
- داشبورد مدیریتی با چند نقش کاربری (مدیر / مربی / ورزشکار)

تمام چیز **سمت کلاینت** است — هیچ بک‌اند/سروری وجود ندارد. داده‌ها در `localStorage` مرورگر
ذخیره می‌شوند. مناسب برای دمو/پروژه‌ی دانشجویی، نه پروداکشن واقعی.

## ۴. جریان صفحات

```
landingPage  (صفحه عمومی - معرفی باشگاه)
     ↓ دکمه «ورود به پنل» → showLogin()
loginPage    (تب ورود / تب ثبت‌نام)
     ↓ login() موفق → showPage('dashboardPage')
dashboardPage
     ↓ logout() → loginPage
```

صفحات با کلاس `.page` + `.active` کنترل می‌شن (`showPage(id)` در app.js).
داخل dashboard، بخش‌ها با کلاس `.section` + `.active` کنترل می‌شن (`showSection(id, linkEl)`).

## ۶. ماژول «مدیریت اعضا» (مهم‌ترین و جدیدترین بخش)

`memberMgmtSection` با ۳ تب (`switchTab(id, btn)`):

### تب ثبت‌نام (`tabReg`)
فرم کامل: نام، شماره تماس، کد ملی، وزن، قد، تاریخ تولد (ShamsiPicker)، آپلود عکس پروفایل
(`mgPhotoFile` → FileReader → dataURL)، نوع عضویت (`mgType`: ورزشکار/مربی/VIP)، پلن تمرینی
(`mgPlan`: پایه/استاندارد/پریمیوم)، نوع اشتراک (`mgSubType`: ماهانه/سه‌ماهه/سالانه)،
تاریخ شروع (ShamsiPicker)، و **پیش‌نمایش خودکار تاریخ انقضا** (`mgExpiryPreview`،
محاسبه با `calcExpiry()` که با `onChange` تاریخ شروع به‌روز می‌شه).
ثبت با `mgRegister()`، پاک کردن فرم با `mgClear()`.

### تب لیست اعضا (`tabMgList`)
جدول با جستجو (`mgListSearch`) و فیلتر وضعیت (`mgListFilter`: همه/فعال/نزدیک‌انقضا/منقضی).
وضعیت رنگی با `getMemberStatus(m)`:
- 🟢 فعال (`b-active`)
- 🟡 نزدیک انقضا تا ۷ روز (`b-soon`, `NEAR_EXPIRY_DAYS=7`)
- 🔴 منقضی (`b-expired`)

دکمه ویرایش → `openEditModal(id)` (مودال `editMemberModal` با فیلدهای `em*`)،
دکمه حذف → حذف از `gymMembers`. رندر با `renderMgList()`.

### تب پروفایل (`tabProfile`)
جستجوی عضو → `showProfileDetail(id)`: نمایش کامل اطلاعات + بج وضعیت + تاریخچه پرداخت‌ها
(از `gymPayments` فیلتر شده با `memberId`) + دکمه «تمدید اشتراک» (`renewSubscription(id)`
که `startDate` رو با امروز جایگزین می‌کنه و expiry رو دوباره حساب می‌کنه).

## ۷. تقویم شمسی (ShamsiPicker) — تازه بازطراحی شده

کلاس `ShamsiPicker` در app.js (خط ~۴۳۸). یک popup با:
- شبکه‌ی روزهای ماه (RTL، شنبه تا جمعه)
- **دو دکمه/درآپ‌داون سفارشی برای انتخاب مستقیم ماه و سال** (به‌جای `<select>` بومی —
  چون select بومی با فونت فارسی داخل باکس کوچک درست رندر نمی‌شد و "..." نشون می‌داد).
  این درآپ‌داون‌ها با کلاس‌های `.sp-dd`, `.sp-dd-btn`, `.sp-dd-list`, `.sp-dd-item` ساخته شدن.
- دکمه‌های ‹ › برای جابه‌جایی ماه قبل/بعد

`SP_YEARS` بازه‌ی ۱۱۰ ساله (۱۰ سال آینده تا ۱۰۰ سال قبل از سال جاری شمسی) رو می‌سازه.
پیکرهای فعال در پروژه: `mgStartDate`, `mgBirthDate`, `emStartDate` (مودال ویرایش), `payDate`
(فرم پرداخت). همه از `initPickers()` ساخته می‌شن. پیکرهایی که `onChange` دارن
(`mgStartDate`, `emStartDate`) باعث آپدیت خودکار پیش‌نمایش انقضا می‌شن.

## ۱۰. محدودیت‌های فعلی پروژه

- **بدون بک‌اند** — همه چیز localStorage، یعنی داده فقط روی همون مرورگر/دستگاهه و با پاک کردن
  کش از بین می‌ره. اگه قراره چندکاربره/واقعی بشه، باید بک‌اند اضافه شه (خارج از scope فعلی).
- فایل‌های `app.js` و `style.css` بزرگ و تک‌فایلی هستن (بدون ماژول/bundler). هر ادیت باید با
  `grep`/`Read` دقیق محل کد رو پیدا کنه.
- تصاویر پروفایل به‌صورت base64 dataURL داخل localStorage ذخیره می‌شن → محدودیت حجم
  localStorage (معمولاً ~۵-۱۰MB) رو در نظر بگیر اگه قراره عکس زیاد اضافه شه.
- پروژه گیت ندارد (not a git repo) — تغییرات نسخه‌بندی نمی‌شن.

## ۱۱. سبک کدنویسی / قراردادها (مهم برای ادامه‌ی کار)

- **کامنت‌ها فارسی** — کوتاه و فقط برای بخش‌های غیربدیهی.
- توابع helper برای localStorage به شکل `getX`/`saveX` (arrow function، یک‌خطی).
- ثابت‌های label/class به شکل آبجکت مپ (مثل `TYPE_LABELS`, `STATUS_CLASS`, `SUB_LABELS`) —
  این الگو رو برای هر enum جدید رعایت کن.
- توابع رندر جدول/لیست معمولاً اسمشون `render*` و یک‌بار در نقاط مناسب صدا زده می‌شن
  (بعد از ثبت/ویرایش/حذف).
- استایل‌ها در style.css با بخش‌بندی کامنت `════════════════` جدا شدن — استایل جدید رو
  در بخش مرتبط (یا انتهای فایل با کامنت بخش جدید) اضافه کن.
- از `direction: rtl` و کلاس‌های آماده‌ی badge (`b-active`, `b-soon`, ...) استفاده کن، چیز
  تکراری نساز.

2. حذف کامل بخش مستقل «افزودن عضو» (سایدبار + section + توابع مرتبط) چون داخل
   «مدیریت اعضا» این قابلیت موجوده.
3. بازطراحی تقویم شمسی (`ShamsiPicker`): جایگزینی `<select>` بومی ماه/سال (که باعث رندر
   غلط متن فارسی "..." می‌شد) با درآپ‌داون سفارشی div-based — انتخاب مستقیم ماه/سال بدون
   فشار مکرر دکمه‌ها.

## ۱۵. نکات رفتاری کاربر (برای تعامل بهتر)

- کاربر فارسی‌زبانه و ترجیح می‌ده توضیحات/پاسخ‌ها هم فارسی باشه.
- وقتی می‌گه «فقط همینو حذف کن، چیز دیگه‌ای تغییر نکنه» → دقیقاً محدود به همون درخواست
  عمل کن، ولی اگه یه reference شکسته (مثل متن یه دکمه که به بخش حذف‌شده اشاره می‌کنه)
  پیدا کردی، اون رو هم اصلاح کن (لازمه‌ی صحت، نه scope creep).
- قبل از گفتن «تموم شد»، اگه preview در دسترسه، توی مرورگر واقعاً تست کن (نه فقط با کد خوندن).
